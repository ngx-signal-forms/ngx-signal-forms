# Field States (archived reference)

This state-visualization demo is **no longer exposed as a standalone route**.

The concepts it taught are still important, but they are now shown in active examples with more realistic context:

- debugger panels in `toolkit-core/error-display-modes`
- long-form debugger output in `form-field-wrapper/complex-forms`
- submission/status behavior in `advanced-scenarios/submission-patterns`

## Concepts still worth keeping from this folder

- `touched` / `dirty` / `valid` / `invalid` / `pending`
- why invalid forms should not always disable submit buttons
- how debugger output helps explain hidden-vs-visible errors

## Prefer these live examples instead

- `toolkit-core/error-display-modes`
- `form-field-wrapper/complex-forms`
- `advanced-scenarios/submission-patterns`
