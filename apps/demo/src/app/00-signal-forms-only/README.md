# Signal Forms Only (archived reference)

This folder is kept as a historical baseline showing what Angular Signal Forms looks like without the toolkit, but it is **no longer part of the live demo route graph**.

## Why it is archived

The live demo now starts with `01-getting-started/your-first-form` so developers land on the smallest recommended toolkit setup instead of a deliberately boilerplate-heavy baseline.

## What this archived example is still useful for

- understanding manual `aria-invalid` / `aria-describedby` wiring
- seeing how much template boilerplate pure Signal Forms requires
- explaining why the toolkit exists to teammates or stakeholders

## Where to go instead

- **Recommended starting point:** `01-getting-started/your-first-form`
- **Strategy-focused follow-up:** `02-toolkit-core/error-display-modes`
- **Full wrapper-based forms:** `04-form-field-wrapper/complex-forms`

If you need the raw baseline implementation details, keep this folder as a reference — just do not treat it as the current onboarding path.
